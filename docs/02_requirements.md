# 02. 要件定義 (Requirements)

## 1. 機能要件
本システムが備える主要な機能は以下の通りです。

| 機能名 | 概要 |
| --- | --- |
| **QRコード自動検出** | OpenCVを利用し、カメラに映ったQRコードをリアルタイムで検知・デコードする。 |
| **打刻判定ロジック** | 読み取ったIDと当日の打刻状況を照合し、初回なら「出勤」、二回目なら「退勤」として自動的に記録する。 |
| **二重打刻防止** | 打刻後の数秒間、または既に退勤済みのユーザーに対しては、意図しない重複打刻を無視する。 |
| **ユーザー管理** | 管理者画面からのユーザー一覧確認、および新規登録（QRコード自動生成を含む）機能。 |
| **履歴確認・集計** | 日付ごとの出退勤情報（ペアリング済みデータ）および、生の打刻ログの閲覧。 |

## 2. 非機能要件
実用性と保守性を考慮した設計目標です。

| 項目 | 概要 |
| --- | --- |
| **実行環境** | 外部サーバー・インターネット通信不要。ローカルのPython/SQLite環境のみで動作。 |
| **ユーザビリティ** | 作業員（ユーザー）は画面操作を一切行わず、「かざすだけ」で完了するUIレス（Zero UI）設計。 |
| **堅牢性** | 物理的な端末（入り口・出口）を分けることで、システムロジックのみに頼らない確実な動線管理を実現。 |
| **拡張性** | 画像処理部とWebサーバー部を分離し、将来的な顔認証システム等へのアップグレードが容易な構成。 |
