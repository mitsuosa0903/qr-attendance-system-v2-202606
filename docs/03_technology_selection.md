# 03. 技術選定 (Technology Selection)

## 🛠️ 使用技術一覧 (Tech Stack)

| 領域 | 技術 | 選定理由 |
| :--- | :--- | :--- |
| **言語** | **Python 3.10+** | 画像処理（OpenCV）との親和性が高く、スクリプトからWebサーバーまで一気通貫で開発できるため。 |
| **Webアプリ** | **Flask** | 軽量でディレクトリ構成の自由度が高く、今回のようなマルチプロセス/マルチポート構成の構築に適しているため。 |
| **画像解析** | **OpenCV** | リアルタイムな動画ストリーミングと、高度なQRコード検出・矩形描画を低レイテンシで実現するため。 |
| **データベース** | **SQLite3** | ファイルベースで動作するため、小規模環境においてデータベースサーバーの構築・管理コストが不要なため。 |
| **フロントエンド** | **Vanilla JS / HTML / CSS** | シンプルな情報表示がメインであるため、ビルド不要な標準技術のみを採用。UIには一部 Mermaid.js を使用。 |
| **自動テスト** | **pytest / Playwright** | バックエンドのロジック検証に加え、実際のブラウザ操作を通じたE2Eテストで品質を担保するため。 |

## 💡 選定のポイント
* **バックエンド主導のカメラ制御**: ブラウザ側のJavaScript（WebRTC等）ではなくバックエンド（Python）でカメラを制御することで、より低レベルなチューニングや、将来的なAI推論（顔認証）の組み込みを容易にしています。
* **SQLiteの採用**: ポートフォリオとして「手軽にクローンしてすぐ動かせる」ことを重視し、環境構築のハードルを最小限に抑えました。
