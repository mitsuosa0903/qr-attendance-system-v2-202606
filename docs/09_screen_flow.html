画面遷移図 (Screen Transition Diagram)

本システムは、打刻ミス防止のために用途ごとに役割を分離した 3つの独立したFlaskアプリ（ポート） で構成されています。 それぞれのアプリが持つ画面とその遷移、および共通データベースとの関係を以下に示します。

1. 全体俯瞰図

graph TD
    subgraph IN ["【出勤用端末】 Port 5001"]
        IN_Main["入館打刻画面 (/)"] <--> IN_Add["社員登録画面 (/add_user)"]
    end

    subgraph OUT ["【退勤用端末】 Port 5002"]
        OUT_Main["退館打刻画面 (/)"] <--> OUT_Add["社員登録画面 (/add_user)"]
    end

    subgraph ADMIN ["【管理者用画面】 Port 5003"]
        ADMIN_Dash["管理ダッシュボード (/)"]
        ADMIN_Dash --> QR_DL["QRコード再発行 (ダウンロード)"]
    end

    IN_Main    -.->|scan_logs へ記録| DB[(共通DB
attendance.db)]
    OUT_Main   -.->|scan_logs へ記録| DB
    IN_Add     -.->|users へ登録| DB

2. 各画面の詳細

2-1. 出勤・退勤用端末（Port 5001 / 5002）

作業員が「かざすだけ」で操作を完結させるためのシンプルな構成です。

画面名 パス 主な操作・機能 遷移先
打刻メイン / カメラ起動、QRコード検知、打刻結果のリアルタイム表示 社員登録画面へ
社員登録 /add_user 新規ユーザー登録、QRコードの自動生成・ダウンロード 打刻メインへ

2-2. 管理者用画面（Port 5003）

勤怠状況を一元管理するダッシュボードです。

画面名 パス 主な操作・機能 遷移先
管理ダッシュボード / 出退勤の自動集計、生の打刻ログ表示、ユーザー一覧確認 QRコード再発行（ダウンロード）
QRコードDL /qr_download/<user_id> 指定ユーザーのQRコード画像をダウンロード（再発行） —
