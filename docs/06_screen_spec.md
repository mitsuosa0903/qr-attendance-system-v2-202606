# 06. 画面仕様 (Screen Specification)

## 🖥 画面一覧

| 画面名 | パス | 役割 | 主な表示内容 |
| :--- | :--- | :--- | :--- |
| **打刻メイン画面** | `/` | 勤怠の受付 | カメラ映像（リアルタイム）、打刻成功時のステータス表示（名前、時刻） |
| **ユーザー登録画面** | `/add_user` | 社員情報の追加 | ID/氏名入力フォーム、登録完了後のQRコード表示 |
| **管理ダッシュボード** | `/` (Admin) | データの閲覧 | 全ユーザー一覧、本日の出退勤ペア状況、最新50件のスキャン生ログ |

## 📐 各画面の詳細仕様

### 1. 入出力用端末（出勤・退勤アプリ）
* **操作性**: キーボード・マウス操作を前提とせず、QRコードをカメラにかざすだけで処理が完了するように設計。
* **フィードバック**: 打刻成功時は青色（出勤）または赤色（退勤）のメッセージが画面下部に表示されます。
* **リダイレクト**: `/add_user` からの登録完了後は、自動的に打刻待ち画面 `/` に戻ることで連続運用を可能にしています。

### 2. 管理画面
* **一覧性**: `attendances` テーブルのデータを集計済みの状態で表示し、管理者が一目で就業状況（勤務中か退勤済みか）を把握できるようにしています。
* **QR再発行機能**: 既存ユーザーのQRコードをいつでもダウンロードし直せるボタンを設置。
