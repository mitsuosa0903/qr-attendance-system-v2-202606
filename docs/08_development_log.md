# 08. 開発ログ (Development Log)

## 🚧 設計の変遷と課題解決
本プロジェクトの開発過程で発生した技術的課題と、それに対する設計判断の記録です。

### 1. 「1画面」から「3端末分割」への大きな転換
当初は「1つの画面で読み取りも管理も行う」というシンプルな設計でした。しかしプロトタイプ運用の結果、以下の致命的な課題が見つかりました。

* **課題1: 二重打刻の頻発**
  カメラの前にQRを置くと、数ミリ秒単位で「出勤→退勤→出勤...」と連続読み取りされてしまう。
* **課題2: 出退勤判定の複雑化**
  同じ端末で判定しようとすると、「今から入るのか、今から帰るのか」をプログラム側で100%正確に当てるのが困難であった。

### 2. 解決策：物理的な「動線」とシステムの同期
論理的なプログラムだけで解決するのではなく、システムの構成自体を事業所の「入り口（出勤専用）」と「出口（退勤専用）」に物理的に分けて配置する前提へと設計変更しました。

* **メリット**:
  * 入り口のPCで読めば「出勤」と断定でき、出口なら「退勤」と確実。
  * 従業員の移動という「物理的なイベント」に連動させることで、システム負荷と誤作動を激減させた。

### 3. 技術的に工夫した点
* **ステートの一時保存**: 短時間の連続読み取りを無視するクールダウン機能を実装。
* **Motion JPEGによる配信**: ブラウザJSではなくサーバーサイドで映像処理を終えてから配信することで、フロントエンドの軽量化を実現。
